API Documentation

Authentication
Login
POST

"https://www.tourday.team/api/auth/login/"

Send either username or email address with password. If the credentials are valid then the response will contain a key. use that key to acces all data.

{
"username": "asif",
"email": "asif.qubit@mail.com",
"password": "******"
}
Sign up
POST

"https://www.tourday.team/api/auth/signup/"

If the credentials are valid then the response will contain a key. use that key to acces all data.

{
"username": "asif",
"email": "asif.qubit@mail.com",
"password": "******"
}
Forget Password
POST

"https://www.tourday.team/api/forget_password/"

Use username or email address in username_email field. Use slug from response in reset_password url.

{
"username_email": asif.qubit@gmail.com
}
Response:
{
"status": 200,
"slug": "asif"
}
Reset Password
POST

"https://www.tourday.team/api/reset_password/username/"

Use code that sent to your email account. Use slug from forget_password response in reset_password url instead of username. Password must be greater than 7 characters.

{
"code": TKPHHKGC,
"password1": ********,
"password2": ********
}
Delete Account
POST

"https://www.tourday.team/api/delete_account/"

Send a POST request with user token the account will be deleted.

Response:
{
"status": 200
}

GET

"https://www.tourday.team/api/profile/"

Token is required. Add token with request header.

}

},
"is_completed": true
}
Update Profile Info
POST

"https://www.tourday.team/api/profile/"

Token is required. Add token with request header. Send only specific filed to update it.
Ex: If you want to update only name filed, just send name in the request. Keep other field blank.

{
"name": "Asif Shahriar",
"email": "asif.qubit@gmail.com",
"fb": "asif.8617",
"insta": "_asif_shahriar",
"city": "Rangpur, Bangladesh",
"bio": "Full Stack web developer",
"picture": picture.jpg
}
Get User Posts
GET

"https://www.tourday.team/api/get_posts/username"

Send username with the url and get the response containing all posts. If that user has more than 10 posts, you will get next page url in "next" in the response.

{
"count": "post count",
"next": "next page url",
"previous": "previous page url",
"results":
{
"id": 1,
"post": "post here"
"image": "/media/user/pic.jpg",
"date": "2020-09-16",
"location": "Dhaka",
"user": 1,
"liikes": [1,2,3]
}
}
